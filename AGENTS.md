# AGENTS.md — 给 AI 协作者（codex / Claude 等）

> 本项目由人类作者主导，多个 AI 助手协作。进来先读这份，理解分工与铁律，别各干各的。
> Claude 那边的同源约定见 `00_项目总览/AI协作方法论.md` 与 `dongbeiya-city/CLAUDE.md`。

## 这是什么

「旅顺口写作计划」——《临高启明》衍生的历史架空创作项目，两条产出线：
- **写作线**：小说正文 + 人物/组织 + 故事线 + 设定考据（本目录 `00_`–`90_`）。
- **游戏线**：`dongbeiya-city/`（Godot 城建模拟，详见其 `CLAUDE.md`）。
- **短剧/视频线**（新）：用现成 AI 模型链把小说做成短剧，见 `06_视觉与视频素材/旅顺口短剧制作工作流.md`。

## 分工

- **正文写作**：作者主导，AI 据设定辅助。
- **UI / 视觉**：codex 负责视觉呈现；Claude 只写交互逻辑、视觉留占位符。两边别互相覆盖。
- **游戏代码 / 结构性文档**：Claude 侧维护（CLAUDE.md、设定→数据映射表）。

## 铁律（所有 AI 都要守）

1. **设定是唯一真理之源，单向流动**：设定 → 正文 / 游戏数据 / 提示词。不要在正文、代码或提示词里临时编设定；要改设定，先改 `02_人物与组织/`、`03_故事线专项/`、`04_设定与游戏设计/` 的源文档。
2. **不要制造重复的真理源**。已知隐患：`04_设定与游戏设计/007.md` 与 `旅顺口游戏设定文档.md` 内容近乎重复（两份主设定文档），**未合并前以 `旅顺口游戏设定文档.md` 为准**，不要再 fork 第三份。
3. **副本漂移警告**：本项目可能同时存在 iCloud 与 `Desktop/旅顺口写作计划/` 两份。动手前确认在哪份上操作，避免改了一份另一份过期。
4. **改动前先指路、后改**：明确读哪个文件再动，不要全局猜测式大改。
5. 新建文档先确认归属目录、不与现有文件重复。
