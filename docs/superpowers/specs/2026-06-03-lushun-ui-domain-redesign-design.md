# 旅顺口 UI 领域准确全页面翻新设计

## 目标

将 `07_UI原型/ui-demo` 的 7 个静态 HTML 页面，从泛现代策略 HUD 改成完整的“崇祯十三年·旅顺口·元老院军政工业飞地”游戏界面。重点不是换色，而是把抽象图形替换为符合设定的具体军政、筑城、外交、科技、建造对象。

## 视觉原则

- 冷冬军事工程感：冷灰、铁黑、羊皮纸、钝金、砖红、海军蓝，少量暖橙作为灯火和军令焦点。
- 去泛科幻 HUD：减少亮青发光、玻璃面板、纯网格装饰和 emoji 占位符。
- 对象化主视觉：每页的中部主区域必须是世界观内的具体对象，而不是抽象图表。
- 保留策略密度：保留顶部状态、左右信息栏、底部命令栏和原有数据/交互脚本。

## 页面对象

- `index.html`：金州地峡值班军令台，背景图作为战役地图，路线像军务标注。
- `bastion-section.html`：真实棱堡剖切/工程图，包含外墙、土垒、炮位夹层、守军库、地基暗道。
- `strategic-map.html`：东北亚海陆态势军图，带海路、威胁箭头、罗盘和情报标牌。
- `map-builder.html`：寒地施工测绘网格，地块像冻土工地和木桩放样。
- `senate-roster.html`：元老卷宗桌面，属性、命运、关系像档案册与印章。
- `diplomacy.html`：外交案牍桌面，势力关系像条约、密报、蜡封和筹码。
- `tech-tree.html`：研究署蓝图板，节点像卷宗卡片，连线像工程图纸。

## 验证标准

- 所有 HTML 页面引用 `ui-kit.css`。
- `ui-kit.css` 定义新材质变量和通用组件。
- 每页包含本页领域准确主视觉标记。
- 页面源码不再使用 emoji/pictographic glyph 作为功能图标。
- 静态截图可以通过 Playwright 生成，无页面脚本错误。
