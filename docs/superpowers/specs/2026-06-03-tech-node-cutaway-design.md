# 科技节点对象剖面设计

## 目标

在 `tech-tree.html` 中引入“节点展开 = 技术对象剖面”的交互。用户点击科技节点后，右侧详情不只显示成本、风险和效果，还要展开该技术成立所依赖的层级结构，使科技树像棱堡剖面一样成为可读的工业对象。

## 首个节点

以“黑火药标准化”为首个完整样例。展开层级为：

1. 原料层：硝石、硫磺、木炭与纯化来源。
2. 配方层：配比、湿混/干混、批次稳定。
3. 工艺层：研磨、筛分、压饼、造粒、干燥、防潮。
4. 检验层：燃速、残渣、炮膛压力、试爆场。
5. 包装运输层：火药桶、封蜡、批号、仓储湿度。
6. 军事应用层：火枪药、炮药、地雷药、信号药分级。
7. 扩散管控层：掌握技师、泄露路径、敌方复制时间。

## 交互

- 点击节点后，右侧详情区显示 `tech-cutaway` 分层视图。
- 每一层是可读卡片，带层号、标题、依赖说明和状态。
- 默认节点可使用通用剖面；黑火药标准化必须有专属剖面。
- 仍保留现有成本、状态、扩散风险、解锁效果、行动按钮。

## 验证

- 静态验证脚本检查 `tech-tree.html` 包含 `tech-cutaway`、`tech-layer-stack`、`dependency-layer`。
- 静态验证脚本检查源码包含 `cutaway` 数据，并包含“黑火药标准化”的七层关键文本。
